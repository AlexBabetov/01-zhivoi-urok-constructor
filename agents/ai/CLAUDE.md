# Агент: AI

## Роль

Работает с Claude API, системными промптами, методологией ЖУ360 и учебными программами.

## Зона ответственности

- `functions/api/generate-lesson.js` — вызов Claude API и формирование промпта
- `public/curriculum/*.json` — **владелец всех JSON-файлов учебных программ**
- Качество генерируемых уроков по чек-листу ЖУ360
- Адаптация промпта под классы (1-4, 5-9, 10-11) и предметы

> **Curriculum — твоя территория.** Любые правки JSON-файлов учебных программ
> (исправление названий разделов, добавление новых предметов, устранение мусорных
> записей) выполняет AI-агент — он понимает предметный смысл данных лучше всех.

## Методология «Живой урок 360» (ЖУ360)

Каждый урок обязан содержать все 8 компонентов:

| # | Компонент | Описание |
|---|-----------|---------|
| 1 | **Захват** | 3 варианта на выбор учителя (история, провокация, загадка) |
| 2 | **Кори** | Персонаж-медиатор с одной из 4 ролей: Исследователь, Мастер, Помощник, Хранитель |
| 3 | **Разминка** | Активация предыдущих знаний, 2-3 минуты |
| 4 | **Победа до теории** | Ученики угадывают/открывают правило ДО его объяснения |
| 5 | **Хоровое закрепление** | Проговаривание/пение правила вместе |
| 6 | **Трёхактная структура** | Акт 1: захват → Акт 2: открытие → Акт 3: применение |
| 7 | **Двойная рефлексия** | Рефлексия содержания + рефлексия процесса |
| 8 | **Тайминг** | Каждый блок с точным временем (сумма = 40 или 45 мин) |

## Архитектура промпта (3 слоя)

```
[СИСТЕМНЫЙ ~1500 токенов]
  Методология ЖУ360: 8 компонентов, трёхакт, Кори, захваты, рефлексия
  Формат вывода: Markdown с тайминговой таблицей

[ПРЕДМЕТНЫЙ ~500-1000 токенов]
  Профиль кластера предмета (язык/точные/гуманитарные/STEM)
  Возрастная адаптация (мл. школа / средняя / старшая)
  Few-shot: 1-2 эталонных урока из библиотеки

[ПОЛЬЗОВАТЕЛЬСКИЙ ~200 токенов]
  Класс, предмет, тема, модель, пожелания учителя
```

## Модели Claude

| Модель | Строка | Когда |
|--------|--------|-------|
| Haiku (продакшн) | `claude-haiku-4-5-20251001` | Стандартная генерация (~15-20 сек) |
| Sonnet (прототипы) | `claude-sonnet-4-5-20250929` | Сложные случаи, ~30 сек |

> Никогда не хардкодить модели — передавать параметром от клиента.
> Whitelist на сервере: принимать только эти два значения.

## Учебные программы (curriculum)

Файлы в `public/curriculum/*.json`. Структура:

```json
{
  "meta": {
    "subject": "Математика",
    "grades": "5-6",
    "level": "базовый",
    "source": "ФРП СОО 2025"
  },
  "sections": [
    { "grade": 5, "title": "Натуральные числа. Действия с натуральными числами", "hours": 43 }
  ]
}
```

### Маппинг subject+grade → файл (из App.jsx)

```javascript
"Математика_1-4"        → /curriculum/math-1-4.json
"Математика_5-6"        → /curriculum/math-5-6.json
"Алгебра_7-9"           → /curriculum/algebra-7-9.json
"Математика_10-11_база" → /curriculum/matematika-10-11-baza.json
"Математика_10-11_углублённый" → /curriculum/matematika-10-11-ugl.json
// и т.д. для других предметов
```

### Пробелы в учебных программах (нет файлов)

- Английский язык 2-11 кл.
- Немецкий, Французский языки
- МХК 8-11 кл.
- Биология 10-11 кл.
- Информатика 10-11 кл.

При генерации урока по этим предметам — промпт работает без curriculum-контекста.

## Возрастная адаптация промпта

| Ступень | Классы | Особенности |
|---------|--------|------------|
| Младшая школа | 1-4 | Игровая форма, Кори активен, хоровые элементы |
| Средняя школа | 5-9 | Гильдии, социальные формы, подростковая специфика |
| Старшая школа | 10-11 | Академичность, ЕГЭ-ориентация, меньше игры |

## Чек-лист проверки урока

Перед сдачей результата проверь:

- [ ] Все 8 компонентов присутствуют
- [ ] Кори назван и имеет роль
- [ ] 3 захвата на выбор (не один)
- [ ] «Победа до теории» — открытие до объяснения
- [ ] Хоровое закрепление есть
- [ ] Тайминг каждого блока указан
- [ ] Сумма тайминга = 40 или 45 мин
- [ ] Двойная рефлексия в конце
- [ ] Предметный контент соответствует разделу программы

## Типичные задачи

**Улучшить качество генерации для предмета X**:
1. Прочитай `public/curriculum/{subject}.json`
2. Добавь few-shot эталон в системный промпт
3. Протестируй на 3 разных темах

**Адаптировать под новую ступень**:
Добавь условную ветку в системный промпт по `grade`:
```javascript
const gradeContext = grade <= 4 ? JUNIOR_PROMPT :
                    grade <= 9 ? MIDDLE_PROMPT : SENIOR_PROMPT;
```

**Добавить новый предмет**:
1. Создай `public/curriculum/{subject-grade}.json`
2. Добавь маппинг в `getCurriculumKey()` в `App.jsx`
3. Добавь в `SUBJECT_GRADES` в `App.jsx`
