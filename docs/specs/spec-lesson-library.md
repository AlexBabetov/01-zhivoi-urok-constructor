# Библиотека уроков: index.json + публичный просмотр — Feature Spec

**Фаза:** Sprint 2 (март 2026)
**Приоритет:** 🟡 Важно
**Статус:** Draft — март 2026

---

## Проблема

### Что сломано прямо сейчас

`useLessons()` в App.jsx вызывает `/api/lessons`, но этого маршрута **не существует** — нет ни Netlify Function, ни redirect в `netlify.toml`. Все запросы к индексу урока падают в SPA-хendler (→ `index.html`). Библиотека уроков у пользователей **не загружается**.

### Что отсутствует по плану

1. **Нет публичного доступа.** `LibraryView` требует авторизованного `user`. Учитель без аккаунта, методист, родитель — не могут посмотреть ни один урок без регистрации.
2. **Нет публичного URL.** Библиотека открывается только как модал внутри конструктора. Поделиться ссылкой на конкретный урок или на библиотеку в целом невозможно.
3. **Нет функции `/api/lessons`.** `save-lesson.js` корректно пишет `public/lessons/index.json` в GitHub, но механизма его чтения нет.
4. **Фильтрация по автору на клиенте.** Учитель видит только свои уроки через `author_id === user?.id` — но раз индекс не грузится, это неважно.

**Для кого проблема:** учитель (не видит свою библиотеку), методист (не может просмотреть накопленные уроки), завуч (Sprint 4 ждёт данных рефлексий, которые не записываются из-за пустой библиотеки).

---

## Решение

Три связанных изменения:

1. **Netlify Function `lessons.js`** — читает `index.json` и отдельные файлы уроков из GitHub API. Маршрут `/api/lessons` начинает работать.
2. **Публичный режим `LibraryView`** — гость (без логина) видит все уроки, но не видит email авторов. Фильтр «только мои» появляется только у авторизованных учителей.
3. **Публичный URL `/library`** — React Router маршрут, открывает `LibraryView` без модала. Можно сохранить в закладки и поделиться ссылкой.

---

## User Stories

- Как **учитель**, я хочу открыть библиотеку и увидеть список своих уроков, чтобы быстро найти нужный план без повторной генерации.
- Как **учитель**, я хочу переключиться на «Все уроки», чтобы посмотреть, что генерируют коллеги, и взять идею.
- Как **методист**, я хочу зайти на `urok360.koriphey.ru/library` без логина, чтобы посмотреть накопленную библиотеку и оценить качество уроков.
- Как **завуч**, я хочу видеть в карточке урока отметку «Проведён» и оценку учителя, чтобы понимать, какие уроки реально используются (Sprint 4).
- Как **разработчик**, я хочу, чтобы `index.json` обновлялся атомарно при каждом сохранении, чтобы данные в библиотеке всегда были актуальны.

## Требования

### Must Have (MVP)

- [ ] **`netlify/functions/lessons.js`** — новая функция с двумя режимами:
  - `GET /api/lessons` — возвращает `public/lessons/index.json` из GitHub (массив записей)
  - `GET /api/lessons?file=subject/grade/filename.json` — возвращает полный JSON урока
- [ ] **`/api/lessons` redirect** в `netlify.toml` → `/.netlify/functions/lessons`
- [ ] **Публичный доступ к индексу** — `lessons.js` не требует Authorization-заголовка, любой может читать список
- [ ] **Скрытие email** — поле `author_email` не возвращается гостям (только `author_id`)
- [ ] **`LibraryView` без `user`** — если `user === null`, компонент рендерится, показывает все уроки, скрывает кнопку «Провёл урок» (рефлексия только для авторизованных)
- [ ] **React Router маршрут `/library`** — `<Route path="/library" element={<LibraryPage />} />`; при прямом заходе библиотека открывается полноэкранно, не как модал
- [ ] **Кнопка «Библиотека»** в шапке доступна для гостей (сейчас скрыта за auth)

### Should Have

- [ ] **Фильтр «Мои уроки / Все уроки»** — переключатель в шапке `LibraryView`, доступен только авторизованным. По умолчанию — «Мои уроки»
- [ ] **Кэш на клиенте** — второй вызов `useLessons()` за 60 сек не делает fetch, берёт из памяти
- [ ] **Permalink на урок** — URL вида `/library?id=русский-язык-2-1-безударные-гласные-haiku` открывает конкретный урок напрямую
- [ ] **`lessons.js` timeout** в `netlify.toml` = 10 сек (GitHub API обычно быстрый)

### Won't Have (v1)

- Пагинация (библиотека < 200 уроков — умещается в один JSON)
- Комментарии к урокам от коллег
- Рейтинг / лайки уроков
- Публикация `index.json` как GitHub Pages (уроки читаются через API-функцию, не raw.githubusercontent.com — иначе нужен деплой после каждого сохранения)

## Технические детали

### 1. Новая функция `netlify/functions/lessons.js`

```js
// GET /api/lessons           → читает public/lessons/index.json из GitHub
// GET /api/lessons?file=...  → читает public/lessons/{file} из GitHub

const GITHUB_TOKEN = process.env.GITHUB_TOKEN;
const REPO = process.env.GITHUB_REPO || "AlexBabetov/01-zhivoi-urok-constructor";
const BASE = `https://api.github.com/repos/${REPO}/contents`;

// Вспомогательная функция getFile — такая же как в save-lesson.js
// Возвращает { content, sha } или null если 404

// Режим 1 (индекс): stripEmails(indexArr) — убираем author_email из каждой записи
// Режим 2 (файл урока): отдаём полный JSON без изменений (email автора уже нет в lesson.json,
//   только author_id — UUID Supabase, не PII)
```

Аутентификация: **не требуется** — функция публичная. GITHUB_TOKEN используется только для чтения (GitHub API rate limit: 60 req/час без токена vs 5000 с токеном — нужен токен для надёжности).

### 2. `netlify.toml` — добавить redirect

```toml
[[redirects]]
  from = "/api/lessons"
  to = "/.netlify/functions/lessons"
  status = 200
```

Добавить в `[functions]`:
```toml
[functions."lessons"]
  timeout = 10
```

### 3. React Router — `/library` маршрут

Сейчас App.jsx использует условный рендеринг без Router. Если добавлять настоящие маршруты — большой рефакторинг. Более лёгкий путь: **читать `window.location.pathname` при старте** и если `/library` — сразу открывать `LibraryView` на весь экран вместо конструктора. Не трогаем Router.

```js
// App.jsx — в самом начале компонента App:
const startInLibrary = window.location.pathname === "/library";
const [showLibrary, setShowLibrary] = useState(startInLibrary);
```

Permalink на урок: `?id=` читается из `window.location.search` и передаётся в `LibraryView` как `initialId`.

### 4. Изменения в `LibraryView`

```jsx
// До:
function LibraryView({ onClose, user }) {
  const lessons = isAdmin ? allLessons : allLessons.filter(l => l.author_id === user?.id);

// После:
function LibraryView({ onClose, user }) {
  const [showMine, setShowMine] = useState(!!user); // гость видит все
  const lessons = (user && showMine)
    ? allLessons.filter(l => l.author_id === user.id)
    : allLessons;
  // isAdmin — видит все всегда
```

Кнопка «Провёл урок» (`ReflectionModal`) → скрыта если `!user`.

## Критерии приёмки

- [ ] `GET /api/lessons` без Authorization возвращает массив записей, `author_email` отсутствует
- [ ] `GET /api/lessons?file=русский-язык/2/1_безударные-гласные_haiku.json` возвращает полный урок
- [ ] Открыть `urok360.koriphey.ru/library` в режиме инкогнито → библиотека загружается, кнопка «Провёл урок» скрыта
- [ ] Авторизованный учитель открывает библиотеку → видит только свои уроки по умолчанию, может переключиться на «Все»
- [ ] После сохранения нового урока → через 3 сек открыть `/api/lessons` → новая запись в индексе
- [ ] Проверить на мобильном устройстве (iOS Safari / Android Chrome): карточки уроков читаемы, скролл работает

## Метрики успеха

**Количественные:**
- 0 ошибок 404 на `/api/lessons` (сейчас 100%)
- Время загрузки индекса < 1.5 сек (GitHub API → Netlify Function → клиент)
- Библиотека открывается у гостей — 100% запросов к `/library` отдают контент

**Качественные:**
- Учителя на пилоте говорят «могу быстро найти старый урок» (сейчас — не могут)
- Методист может прислать ссылку на конкретный урок в мессенджер

## Зависимости и риски

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| GitHub API rate limit (60 req/ч без токена) | Средняя | GITHUB_TOKEN уже есть в env; функция использует его |
| `index.json` рассинхронизируется при ошибке `save-lesson.js` | Низкая | `save-lesson.js` уже атомарно пишет оба файла, откатиться нельзя — проверить логику retry |
| Большой `index.json` (> 1000 уроков) замедляет загрузку | Очень низкая (сейчас < 50 уроков) | В будущем — пагинация по subject/grade |
| CRA не поддерживает `<Route>` без Router | Низкая | Используем `window.location.pathname` без Router (см. п.3) |
