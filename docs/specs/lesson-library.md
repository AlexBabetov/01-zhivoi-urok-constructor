# Библиотека уроков — Feature Spec

**Фаза:** Sprint 2
**Приоритет:** 🟡 Важно
**Статус:** Draft

---

## Проблема

Сейчас сгенерированный урок существует только в браузере — после закрытия вкладки он пропадает навсегда. Учитель не может вернуться к хорошему уроку, методист не может видеть что генерирует команда, нет накопления базы. Пилот по русскому языку (март 2026) требует, чтобы хотя бы 10+ уроков можно было просмотреть повторно и оценить.

---

## Решение

Добавить в конструктор кнопку «Сохранить урок» — она отправляет JSON урока в Netlify Function, которая коммитит файл в GitHub репозиторий в папку `lessons/`. Рядом поддерживается `lessons/index.json` — плоский список всех сохранённых уроков. Отдельная страница «Библиотека» в UI показывает этот список и позволяет открыть любой урок.

---

## User Stories

- Как **учитель**, я хочу нажать «Сохранить» после генерации, чтобы урок не пропал и я мог вернуться к нему завтра
- Как **учитель**, я хочу открыть библиотеку и найти свой прошлый урок по теме или предмету, чтобы использовать его снова или доработать
- Как **методист**, я хочу видеть все сохранённые уроки команды, чтобы оценить качество генерации и собрать обратную связь
- Как **методист**, я хочу знать какие уроки уже есть в библиотеке, чтобы не дублировать генерацию

---

## Требования

### Must Have (MVP)
- [ ] Кнопка «Сохранить урок» на экране результата (шаг 4)
- [ ] Netlify Function `save-lesson`: принимает JSON урока, коммитит в GitHub через API
- [ ] Файл урока: `lessons/{subject}/{grade}/{lesson_num}_{slug}.json`
- [ ] `lessons/index.json` обновляется при каждом сохранении (subject, grade, lesson_num, topic, model, saved_at, filename)
- [ ] Страница «Библиотека» в UI: список уроков из index.json с фильтром по предмету и классу
- [ ] Клик на урок — открывает его в том же UI (экран результата, без повторной генерации)

### Should Have
- [ ] Уникальное имя файла гарантирует отсутствие дублей (по subject+grade+lesson_num)
- [ ] Кнопка «Обновить» — перезаписать урок по тому же ключу (если перегенерировали)
- [ ] Дата и автор (имя учителя) в карточке библиотеки
- [ ] Поиск по теме (простой `filter` по `topic`)

### Won't Have (v1)
- Авторизация и разграничение доступа по учителю — Sprint 3 (Supabase Auth)
- Удаление уроков из UI — только через GitHub напрямую
- Комментарии и оценки уроков — Sprint 3
- Публичный доступ по ссылке — Sprint 3

## Технические детали

**Структура файлов в репо:**
```
lessons/
  index.json              ← плоский массив всех уроков (метаданные)
  русский-язык/
    2/
      109_oduSh-neoduSh.json
  математика/
    3/
      45_drobi.json
```

**`index.json` — структура записи:**
```json
{
  "id": "ru-2-109",
  "subject": "Русский язык",
  "grade": 2,
  "lesson_num": 109,
  "topic": "Одушевлённые и неодушевлённые существительные",
  "model": "Исследование",
  "saved_at": "2026-03-01T10:23:00Z",
  "filename": "русский-язык/2/109_oduSh-neoduSh.json"
}
```

**Netlify Function `save-lesson.js`:**
- Принимает: `{ lesson: {...}, meta: { subject, grade, lesson_num, topic, model } }`
- GitHub API с `GITHUB_TOKEN` (env var, права `contents: write`)
- GET текущего `index.json` → добавить запись → PUT обратно
- PUT файл урока в нужную папку
- Возвращает: `{ ok: true, url: "...github link..." }`

**React (App.jsx):**
- Новая вкладка «📚 Библиотека» в шапке
- `useLessons()` hook: `fetch("/lessons/index.json")`
- `LessonCard` компонент: subject, grade, topic, model, дата
- При открытии урока — `fetch` JSON, рендерить через существующий `LessonResult`

**Переменные окружения (Netlify):**
- `GITHUB_TOKEN` — Personal Access Token, `contents: write`
- `GITHUB_REPO` — `AlexBabetov/01-zhivoi-urok-constructor`

## Критерии приёмки

- [ ] «Сохранить» → файл в GitHub за <5 сек
- [ ] `index.json` обновился
- [ ] Библиотека → урок виден в списке, открывается корректно
- [ ] Тест: Русский язык 2 кл. урок 109, Математика 3 кл.
- [ ] Проверка на мобильном
- [ ] Два урока с одним lesson_num → второй перезаписывает первый
