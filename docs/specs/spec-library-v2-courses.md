# Библиотека v2: Уроки + Авторские курсы — Feature Spec

**Фаза:** Sprint 2–3 (март–апрель 2026)
**Приоритет:** 🟡 Важно
**Статус:** Draft — март 2026
**Зависимость:** Требует завершения spec-lesson-library.md (базовая библиотека уроков)

---

## Проблема

Конструктор накапливает два принципиально разных типа образовательного контента, которые сейчас нигде не живут вместе:

1. **Уроки по программе** — AI-генерированные планы уроков по ФОП (Математика, Русский язык и т.д.), созданные учителями через конструктор.
2. **Авторские курсы и факультативы** — готовые сценарии занятий, разработанные методистами Корифея, прошедшие модерацию. Пример: «Логика Корифей» Пул А (НШ-01…НШ-07, 3–4 класс).

Смешивать их в одном списке нельзя: разная природа (план урока vs методический сценарий), разная аудитория (учитель ищет свой урок vs учитель выбирает курс), разные права (уроки создают учителя, курсы публикует Корифей).

**Для кого проблема:**
- Учитель не может найти готовый курс по логике — его нигде нет в системе.
- Методист Корифея не может опубликовать авторский курс в продукте без разработки.
- Завуч не видит, какие авторские программы используются в школе.

---

## Решение

Двухтабовая библиотека с общим UI-фреймом, но разными источниками данных и разными вьюерами:

```
/library
├── Таб «Уроки» (по умолчанию)      ← AI-генерированные планы учителей
└── Таб «Курсы»                      ← Авторские программы Корифея
```

Курсы хранятся в GitHub в папке `public/courses/`, публикуются редакцией Корифея вручную после модерации. Учитель видит их, читает сценарий занятия и проводит урок — без генерации через AI.

---

## User Stories

- Как **учитель**, я хочу открыть раздел «Курсы» и выбрать «Логика Корифей» для своего 3 класса, чтобы вести факультатив по готовым сценариям без подготовки.
- Как **учитель**, я хочу открыть конкретный модуль (НШ-02 «Закономерности») и читать сценарий занятия прямо в браузере, чтобы не скачивать файлы.
- Как **методист Корифея**, я хочу опубликовать новый модуль курса в GitHub и видеть его в библиотеке без разработки нового UI, чтобы цикл обновления был коротким.
- Как **завуч**, я хочу видеть в дашборде (Sprint 4), сколько учителей ведут «Логику Корифей», чтобы оценить охват авторской программы.
- Как **родитель/гость**, я хочу открыть `urok360.koriphey.ru/library?tab=courses` без логина и посмотреть каталог курсов, чтобы понять, что предлагает Корифей.

### Workflow публикации курса (без разработки)

```
Методист создаёт .md →
Редакционная проверка →
Файл добавляется в public/courses/logika-korifey/ в GitHub →
courses/index.json обновляется вручную (или скриптом) →
Следующий деплой → курс появляется в библиотеке
```

Публикация — **ручная** (pull request в GitHub), не через UI конструктора.

## Разделение: Уроки vs Курсы — ключевые отличия

| Параметр | Уроки (ФОП) | Авторские курсы |
|---------|------------|-----------------|
| **Источник** | AI-генерация учителем | Методисты Корифея |
| **Модерация** | Нет (auto-save) | Редакционная проверка |
| **Формат** | JSON (структурированный план) | Markdown (сценарий + банк материалов) |
| **Структура** | Отдельный урок | Курс → Модуль → N занятий |
| **Хранение** | `public/lessons/` | `public/courses/` |
| **Публикация** | Автоматически при сохранении | Ручной PR в GitHub |
| **Кто видит** | Все уроки публично; «Мои» для авторизованных | Все публично |
| **Рефлексия** | Да (localStorage → Supabase в Sprint 3) | Нет в v1 |
| **Viewer** | `PrimaryResult` / `MiddleResult` (JSX) | Markdown рендерер |

## Технические детали

### 1. Новая функция `netlify/functions/courses.js`

Логика аналогична `lessons.js`:
```js
// GET /api/courses           → читает public/courses/index.json
// GET /api/courses?file=...  → читает public/courses/{file} (md или json)
// Публичная — без Auth
// Возвращает content-type: text/markdown для .md файлов
```

### 2. Изменения в `LibraryView`

```jsx
// Добавить таб-переключатель:
const [activeTab, setActiveTab] = useState(
  new URLSearchParams(location.search).get("tab") || "lessons"
);

return (
  <>
    <TabBar>
      <Tab id="lessons" label="📚 Уроки" />
      <Tab id="courses" label="🧩 Курсы" />
    </TabBar>
    {activeTab === "lessons" && <LessonsTab ... />}
    {activeTab === "courses" && <CoursesTab ... />}
  </>
);
```

### 3. Новый компонент `CoursesTab`

```
CoursesTab
├── CourseGrid (список карточек курсов)
│   └── CourseCard (эмодзи, название, пул, кол-во модулей)
├── CourseDetailView (список модулей)
│   └── ModuleCard (название, занятия, пресквизит)
└── ModuleReader (рендеринг Markdown)
    ├── react-markdown / marked
    └── Кнопка «Скачать PDF» (window.print)
```

### 4. Зависимость: `react-markdown`

```bash
npm install react-markdown
```
Размер: ~35 KB gzip. Альтернатива: `marked` (~20 KB) + `dangerouslySetInnerHTML` — проще, но менее безопасно.

### 5. Загрузка .md файлов в GitHub

Конвертировать уже созданные файлы логики из `/mnt/koriphey-org/` в `public/courses/logika-korifey/`:

```
Logika_Korifey_L-NSH-01_...md → public/courses/logika-korifey/l-nsh-01.md
Logika_Korifey_L-NSH-02_...md → public/courses/logika-korifey/l-nsh-02.md
... и т.д. для НШ-03, НШ-04, НШ-05, НШ-07
```

1. **URL курсов.** Использовать `/library?tab=courses` или сделать отдельный `/courses`? Рекомендация: `?tab=` — проще без Router.
